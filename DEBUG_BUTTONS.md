# 🔍 按钮调试指南

## 问题描述
"生成 PDF" 按钮看不到

## 可能的原因

### 1. 浏览器缓存问题
**解决方法：**
- 按 `Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac) 强制刷新
- 或者按 `F12` 打开开发者工具，右键刷新按钮，选择"清空缓存并硬性重新加载"

### 2. CSS 未加载
**检查方法：**
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 标签，看是否有错误
3. 查看 Network 标签，确认 CSS 文件已加载

### 3. Tailwind CSS 配置问题
**解决方法：**
- 已将 `bg-primary-600` 改为 `bg-emerald-600`（标准 Tailwind 颜色）
- 已将 `bg-gray-700` 改为 `bg-slate-700`（更深的灰色）

### 4. 样式被覆盖
**解决方法：**
- 已添加 `!important` 规则强制应用背景色
- 已添加内联样式确保文字可见

## 调试步骤

1. **打开浏览器开发者工具** (F12)

2. **检查按钮元素**
   - 右键点击按钮应该在的位置
   - 选择"检查元素"
   - 查看 Computed 标签中的实际样式

3. **检查是否有 JavaScript 错误**
   - 查看 Console 标签
   - 如果有红色错误信息，请告诉我

4. **检查按钮是否存在**
   - 在 Elements 标签中搜索 "生成 PDF"
   - 确认按钮元素是否在 DOM 中

5. **检查 z-index 问题**
   - 按钮可能被其他元素遮挡
   - 查看是否有其他元素覆盖在按钮上方

## 临时测试方案

如果按钮仍然不可见，请尝试：

1. **在浏览器控制台运行：**
```javascript
document.querySelectorAll('button').forEach(btn => {
  btn.style.backgroundColor = 'red';
  btn.style.color = 'white';
  btn.style.padding = '20px';
  btn.style.fontSize = '20px';
});
```

2. **检查输出：**
   - 如果按钮变成红色，说明按钮存在但样式有问题
   - 如果没有变化，说明按钮可能没有渲染
